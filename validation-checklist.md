# Validation Checklist

## 运行方式

- 生成 UI 后先按本文件逐项检查。
- 维护仓库时运行 `python scripts/validate_skill.py .`。
- 发布前确认根目录 `SKILL.md` 是中文默认入口，`en-us/` 是同步镜像，`agents/openai.yaml` 与 skill 描述一致。

## 美学匹配

- UI 感觉平静、模块化、圆润且内容优先。
- 强调色冷静、受控并一致使用。
- 大面积表面保持中性且可读。
- 视觉密度匹配页面类型。

## Token 使用

- 颜色使用语义化 tokens。
- 字体使用语义化字号和字重规则。
- 间距、圆角、阴影、边框和动效使用 tokens。
- 暗色模式具备对应 token 行为。
- 暗色主题作用域同时设置 token 覆盖、`color`、`background` 和 `color-scheme`。
- 主题切换不会通过文字色、图标色、控件背景或页面背景 transition 产生低对比中间态。

## 布局语法

- 页面壳遵循选定布局模式。
- 定义 desktop、tablet 和 mobile 布局。
- Section 节奏一致。
- 卡片和面板没有不必要嵌套。
- 除非任务要求，不依赖固定页面顺序。

## 组件配方

- 已选择所需组件。
- 每个组件都有结构、token 依赖和使用规则。
- Button、card、navigation、tag、form、modal、CTA、footer、media、stats 和 plan block 在出现时遵循配方。

## 组件状态

- 有用位置包含 hover 状态。
- 控件包含 active 状态。
- Focus 状态可见。
- Disabled 状态清楚。
- Loading 状态保持布局。
- Error 和 success 状态包含文字或图标含义。
- Empty 状态说明下一步。

## 响应式行为

- 需要时移动端为单列。
- 主控件触控目标至少 44px。
- 导航适配可用宽度。
- 卡片操作可干净重排。
- 长文本不裁切。
- 图片使用稳定 aspect ratio。

## 可访问性

- 文本对比度可读。
- 暗色模式下标题、品牌文字、导航、卡片标题、正文、次级文案、按钮、图标和 focus ring 都可读。
- Focus 可见。
- 支持键盘操作。
- 遵守 reduced motion。
- 表单具备 label。
- 错误消息与字段关联。
- 图片具备合适 alt。
- Heading 层级合理。
- 颜色不是唯一信息载体。

## 内容规则

- 标题简洁。
- 元信息紧凑并弱化。
- CTA 文案短且动作明确。
- 占位文案中性且可替换。
- 不出现虚构私人数据。

## 资产安全

- 不包含私人标志。
- 不包含不可授权图片。
- 不要求复制图标。
- 不出现可识别私人文案。
- 不引入无支持的指标或声明。

## Agent 独立性

- 输出无需隐藏上下文即可使用。
- 所需文件或 section 命名清晰。
- 设计规则可执行。
- 包含验证步骤。

## 发布清洁

- 不出现创建历史或隐藏准备语言。
- 不出现私人名称、域名或可识别资产，除非用户为新任务提供。
- 不存在 JSON 语法错误。
- Markdown 标题清晰有序。
- 示例使用中性占位内容。
- 可访问性和响应式规则完整。
