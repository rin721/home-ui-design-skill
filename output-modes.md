# Output Modes

## 默认模式

用户未指定技术目标时，输出结构化 UI 设计规范。包含视觉方向、tokens、布局、组件、交互、响应式行为、内容规则、可访问性和校验说明。

## UI Design Spec

包含：

- 目标和页面类型。
- 设计语言。
- Token 选择。
- 布局语法。
- 组件列表。
- 交互状态。
- 响应式行为。
- 内容规则。
- 可访问性检查。
- 交付清单。

## HTML / CSS

包含：

- 语义 HTML landmarks。
- token CSS variables。
- 组件 classes。
- 响应式 media queries。
- Focus 和 reduced-motion 样式。
- 中性占位文案。

## React

包含：

- Token CSS variables。
- 组件结构。
- Variants 和 states 的 props。
- 可访问属性。
- 响应式 class 策略。
- Reduced-motion 处理。

## Vue

包含：

- Token CSS variables。
- 需要时使用单文件组件结构。
- 状态 props 和 computed classes。
- 可访问属性。
- 响应式 class 策略。
- Reduced-motion 处理。

## Tailwind

包含：

- Token-to-theme 映射。
- 可复用组件 class patterns。
- 响应式 variants。
- 状态 variants。
- Focus 和 reduced-motion utilities。
- 语义 HTML 说明。

## Figma Prompt

包含：

- Frame 类型和尺寸。
- Token 化颜色和字体方向。
- 布局结构。
- 组件列表。
- 交互标注。
- 响应式 variants。
- 可访问性说明。

## Wireframe Outline

包含：

- 页面 sections。
- 内容层级。
- 组件位置。
- 响应式重排。
- CTA 位置。
- 可访问性要求。

## Component Library Spec

包含：

- 组件清单。
- Variants。

- Token 依赖。
- 状态。
- 可访问性。

- 使用示例。

- Footer。
