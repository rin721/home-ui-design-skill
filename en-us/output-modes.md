# Output Modes

## Delivery Principles

- Explain design decisions before technical details.
- Code deliverables must include token mapping, semantic structure, responsive rules, focus, and reduced motion.
- Dark-mode code must show how the theme scope overrides variables, text color, background, and `color-scheme` together.
- Design specs must include component states, mobile behavior, and accessibility checks, not just visual description.
- Figma prompts, wireframes, and component library specs must use neutral placeholder content and mark replaceable areas.
- When implementation code is not requested, avoid long code blocks. When code is requested, make it directly usable or easy to adapt.

## Default Mode

When no technology target is supplied, output a structured UI design spec. Include visual direction, tokens, layout, components, interactions, responsive behavior, content rules, accessibility, and validation notes.

## UI Design Spec

Include:

- Goal and page type.
- Design language.
- Token selections.
- Layout grammar.
- Component list.
- Interaction states.
- Responsive behavior.
- Content rules.
- Accessibility checks.
- Delivery checklist.

## HTML / CSS

Include:

- Semantic HTML landmarks.
- CSS variables for tokens.
- Component classes.
- Responsive media queries.
- Focus and reduced-motion styles.
- Neutral placeholder copy.

## React

Include:

- Token CSS variables.
- Component structure.
- Props for variants and states.
- Accessible attributes.
- Responsive class strategy.
- Reduced-motion handling.

## Vue

Include:

- Token CSS variables.
- Single-file component structure when useful.
- Props and computed classes for states.
- Accessible attributes.
- Responsive class strategy.
- Reduced-motion handling.

## Tailwind

Include:

- Token-to-theme mapping.
- Reusable component class patterns.
- Responsive variants.
- State variants.
- Focus and reduced-motion utilities.
- Notes for semantic HTML.

## Figma Prompt

Include:

- Frame type and size.
- Tokenized color and typography direction.
- Layout structure.
- Component list.
- Interaction annotations.
- Responsive variants.
- Accessibility notes.

## Wireframe Outline

Include:

- Page sections.
- Content hierarchy.
- Component placement.
- Responsive reflow.
- CTA placement.
- Accessibility requirements.

## Component Library Spec

Include:

- Component inventory.
- Variants.
- Props or configuration.
- Token dependencies.
- States.
- Accessibility.
- Responsive behavior.
- Usage examples.

## Landing Page Structure

Include:

- Navigation.
- Hero.
- Feature cards.
- Proof or stats.
- Product or service details.
- CTA panel.
- Footer.
- Mobile ordering.

## Responsive Layout Plan

Include:

- Desktop layout.
- Tablet layout.
- Mobile layout.
- Breakpoint behavior.
- Navigation changes.
- Card reflow.
- Typography changes.
- Touch target checks.
