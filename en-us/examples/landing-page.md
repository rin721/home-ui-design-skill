# Landing Page Example

## Input Request

Create a landing page for a productivity tool aimed at small teams. Use a calm modular visual language and include design spec output.

## Skill Rules Used

- Cool accent palette.
- Rounded card surfaces.
- Split or compact hero.
- Feature card grid.
- CTA panel.
- Desktop, tablet, and mobile behavior.
- Visible focus and reduced-motion rules.

## Generation Strategy

- Use a compact top navigation with primary links and utility controls.
- Use a hero with one primary claim, one supporting sentence, and two actions.
- Use feature cards with icon tiles and short copy.
- Use a proof or stats row with neutral placeholder metrics.
- End with a rounded CTA panel.

## Structure Draft

1. Navigation.
2. Hero.
3. Feature grid.
4. Workflow section.
5. Stats row.
6. CTA panel.
7. Footer.

## Token Use

- `color.surface.page` for canvas.
- `color.surface.card` for panels.
- `color.brand.primary` for active links, icons, and primary CTA.
- `radius.card.default` for cards.
- `spacing.section.lg` for hero spacing.
- `motion.duration.base` for panels and controls.

## Responsive Notes

- Desktop uses a centered shell and grid feature cards.
- Tablet uses two-column feature cards when width allows.
- Mobile uses a single column, full-width CTAs, and compact navigation controls.

## Accessibility Notes

- One `h1`.
- CTA buttons have visible focus states.
- Icons are decorative unless labeled.
- Motion honors reduced-motion settings.
- Text contrast is checked in light and dark mode.

## Quality Check

- Tokenized color and spacing.
- Reusable card recipes.
- Clear CTA hierarchy.
- Mobile touch targets.
- Neutral placeholder content.
