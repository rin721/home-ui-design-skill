# Content Rules

## Heading Hierarchy

- Use one `h1` per page.
- Use `h2` for major sections.
- Use `h3` for card groups and subsections.
- Keep headings direct and scannable.
- Use strong weight for headings, not decorative type.

## Subtitle Length

- Hero subtitles should be 12 to 28 words.
- Section summaries should be 8 to 22 words.
- Card descriptions should be 8 to 18 words.
- Avoid multi-sentence subtitles unless the page is a long-form article.

## Paragraph Density

- Use short paragraphs for product and landing pages.
- Use 60 to 85 characters per line for long-form content.
- Use line height from 1.6 to 1.75 for article body.
- Separate dense content with subtle dividers or section spacing.

## CTA Copy Length

- Primary CTA: 1 to 4 words.
- Secondary CTA: 1 to 5 words.
- Avoid vague labels such as `Click here`.
- Use verbs that describe the action.

## Information Priority

1. Primary heading.
2. Main benefit or content title.
3. Metadata or context.
4. Short description.
5. Primary action.
6. Secondary links, tags, and utility metadata.

## Module Narrative Order

- Start with context, then value, then details, then action.
- For product pages, move from problem to capability to proof to next step.
- For content indexes, move from item identity to metadata to excerpt to action.
- For profile pages, move from identity to summary to links to detailed sections.
- For archive pages, group by time or category and keep each item compact.

## Empty State Copy

- Use a short heading.
- Explain what is missing.
- Offer one next action when possible.
- Example: `No items yet. Add the first item to start this list.`

## Form Helper Copy

- Place helper text below labels or below fields.
- Keep helper text under 16 words.
- Do not repeat placeholder text as helper text.
- Example: `Use a work email for team access.`

## Error Copy

- State what went wrong and how to fix it.
- Use plain language.
- Keep error copy close to the related field.
- Example: `Enter a valid email address.`

## Placeholder Copy Rules

- Use neutral names such as `Product name`, `Feature title`, `Team member`, and `Plan name`.
- Use neutral values such as `24 items`, `3 steps`, or `Short description`.
- Do not invent recognizable people, companies, domains, or private data.
- Mark placeholder content as replaceable when delivering a spec.

## Metadata Rules

- Metadata should be compact and optionally icon-led.
- Use muted color for dates, categories, reading time, status, and counts.
- Keep metadata groups wrap-friendly.
- Use tags for filtering or classification, not for full sentences.

## CTA Organization

- Use one primary CTA per section.
- Secondary actions should be visually quieter.
- In mobile layouts, stack CTAs or use full-width buttons.
- Avoid placing CTA before the user has enough context.
